# Azure App Service 部署實作說明

本文件提供給 Codex，目標是將目前的 `web_final_Project` 調整成可以部署到單一 Azure App Service 的結構。

專案目前結構：

```text
/project
  /server   ← Express API + SQLite
  /client   ← Vue + Vite
  package.json
```

部署後預期架構：

```text
Azure App Service
  ├─ Express 提供 /api
  ├─ Express 提供 Vue build 後的 client/dist
  └─ SQLite 儲存在 Azure 持久化目錄 /home/data/teamup.sqlite
```

正式環境只使用 Azure 提供的單一 Port，不啟動 Vite dev server。

---

# 一、實作目標

請 Codex 完成以下修改：

1. 根目錄新增正式環境用的 `build` 與 `start` 指令。
2. 保留目前本機開發使用的 `npm run dev`。
3. Vue 前端 build 後產生 `client/dist`。
4. Express 正式環境提供 `client/dist` 靜態檔案。
5. Vue Router 的任意前端路徑重新整理時，不可出現 404。
6. `/api` 路徑仍由 Express API 處理。
7. SQLite 可透過 `TEAMUP_DB_PATH` 指定位置。
8. SQLite 資料夾不存在時，程式應自動建立。
9. Azure 上 SQLite 預期放在 `/home/data/teamup.sqlite`。
10. Express 必須繼續使用 Azure 提供的 `process.env.PORT`。
11. JWT 密鑰必須優先讀取 `process.env.JWT_SECRET`。
12. 不可把 Azure 網址、Port 或密鑰寫死在程式碼內。

# 三、修改 server/app.js

目前 Express 只提供 API，尚未提供 Vue build 後的檔案。

請保留以下行為：

```js
app.use('/api', apiRouter);
```

API 404 必須放在 Vue fallback 之前，避免不存在的 `/api` 被錯誤回傳 Vue 的 `index.html`。

建議結構：

```js
var express = require('express');
var cookieParser = require('cookie-parser');
var logger = require('morgan');
var cors = require('cors');
var path = require('path');
var fs = require('fs');

var apiRouter = require('./routes/api');

var app = express();

app.use(logger('dev'));
app.use(express.json());
app.use(express.urlencoded({ extended: false }));
app.use(cookieParser());

app.use(cors({
  origin: process.env.CLIENT_ORIGIN || true
}));

app.use('/api', apiRouter);

app.use('/api', function(req, res) {
  res.status(404).json({ message: '找不到 API' });
});

var clientDistPath = path.join(__dirname, '../client/dist');
var clientIndexPath = path.join(clientDistPath, 'index.html');

if(fs.existsSync(clientDistPath)){
  app.use(express.static(clientDistPath));
}

app.get('*', function(req, res, next) {
  if(req.path.startsWith('/api')){
    next();
    return;
  }

  if(!fs.existsSync(clientIndexPath)){
    res.status(503).json({
      message: '前端尚未完成建置，請先執行 npm run build'
    });
    return;
  }

  res.sendFile(clientIndexPath);
});

module.exports = app;
```

Codex 可以依照現有 Express 版本調整 fallback 寫法，但必須滿足：

- `/api/*` 走 API。
- 不存在的 `/api/*` 回 JSON 404。
- 非 `/api` 路徑回傳 Vue `index.html`。
- `client/dist` 不存在時，不可讓程式直接崩潰。

注意：目前 server 使用 Express 4，因此 `app.get('*', ...)` 可以使用。

# 五、修改 SQLite 路徑與資料夾建立

目前資料庫路徑為：

```js
var dbPath = process.env.TEAMUP_DB_PATH || path.join(__dirname, 'teamup.sqlite');
```

請保留這個環境變數設計，並補上自動建立父資料夾。

修改 `server/database/db.js` 開頭為類似：

```js
var fs = require('fs');
var path = require('path');
var sqlite3 = require('sqlite3').verbose();

var dbPath = process.env.TEAMUP_DB_PATH ||
  path.join(__dirname, 'teamup.sqlite');

var dbDirectory = path.dirname(dbPath);

if(!fs.existsSync(dbDirectory)){
  fs.mkdirSync(dbDirectory, {
    recursive: true
  });
}

var db = new sqlite3.Database(dbPath, function(err) {
  if(err){
    console.error('SQLite 連線失敗：', err.message);
    return;
  }

  console.log('SQLite database:', dbPath);
});
```

其餘資料表初始化與 CRUD 邏輯應保留。

Azure 環境預期設定：

```text
TEAMUP_DB_PATH=/home/data/teamup.sqlite
```

本機沒有設定 `TEAMUP_DB_PATH` 時，仍使用：

```text
server/database/teamup.sqlite
```

# 六、確認 JWT_SECRET

請搜尋專案內所有：

- `jwt.sign`
- `jwt.verify`
- 寫死的 JWT secret

統一改成讀取：

```js
var jwtSecret = process.env.JWT_SECRET || 'development-only-secret';
```

正式環境不得依賴預設值。

建議在 production 環境未設定時直接中止：

```js
if(process.env.NODE_ENV === 'production' && !process.env.JWT_SECRET){
  throw new Error('正式環境缺少 JWT_SECRET');
}
```

所有 `jwt.sign` 與 `jwt.verify` 必須使用同一個 secret 來源。

不要把真正的 Azure JWT 密鑰提交到 GitHub。

# 七、確認前端 API 設定

目前前端 API client 預期使用：

```js
const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 10000
});
```

這個設計應保留。

正式部署到同一個 App Service 時，不需要設定 `VITE_API_BASE_URL`，讓它預設使用：

```text
/api
```

禁止將以下網址寫死在前端：

```text
http://localhost:3000
http://localhost:5173
```

本機開發由 `client/vite.config.js` 的 proxy 處理：

```js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:3000',
      changeOrigin: true
    }
  }
}
```

這個 proxy 只用於開發，不影響 Azure 正式環境。

# 九、本機正式模式驗證

Codex 完成修改後，請實際執行：

```bash
npm install
npm run install:all
npm run build
npm start
```

預期只需要開啟：

```text
http://localhost:3000
```

不要另外啟動 Vite。

請驗證以下項目：

1. `http://localhost:3000` 能顯示 Vue 首頁。
2. Vue Router 其他頁面可以直接開啟。
3. 在其他 Vue Router 頁面重新整理不會 404。
4. `/api` 仍正常回傳 API 資料。
5. 不存在的 `/api/xxx` 回傳 JSON 404，不是 Vue HTML。
6. 可以註冊與登入。
7. 可以新增、修改與刪除 SQLite 資料。
8. 重新啟動 server 後，SQLite 資料仍存在。
9. 設定自訂 `TEAMUP_DB_PATH` 後，資料庫能在指定位置建立。
10. 未執行 build 時，Express 不應因找不到 `client/dist` 而直接崩潰。

Windows PowerShell 測試自訂 SQLite 路徑：

```powershell
$env:TEAMUP_DB_PATH="$PWD\data\teamup.sqlite"
$env:JWT_SECRET="local-test-secret"
$env:NODE_ENV="production"
npm start
```

Linux/macOS 測試：

```bash
TEAMUP_DB_PATH="$(pwd)/data/teamup.sqlite" \
JWT_SECRET="local-test-secret" \
NODE_ENV=production \
npm start
```


加入：

```text
NODE_ENV=production

JWT_SECRET=<長度足夠的隨機字串>
```


但前後端部署在同一 App Service 時屬於同源，通常不需要額外允許其他 origin。

不要手動設定固定 `PORT`，Azure 會自動提供。

# 十三、SQLite 使用限制

此專案為課堂專題，可以繼續使用 SQLite。

可正常執行：

- 註冊
- 登入
- 新增專案
- 修改專案
- 刪除專案
- 留言
- 收藏
- 一般 CRUD

但需要遵守：

1. App Service instance 保持 1。
2. SQLite 放在 `/home/data/teamup.sqlite`。
3. 不要將正式資料庫放在 repository 目錄。
4. 不要把 Azure App Service 設成唯讀檔案系統。
5. 定期備份 `/home/data/teamup.sqlite`。
6. 不適合大量使用者、高併發或多執行個體。

若之後成為正式大型系統，再改用 Azure SQL 或 PostgreSQL。
