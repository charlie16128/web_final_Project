# TeamUp Campus UI/UX 與創意功能分階段實作計畫書

> 專案：`charlie16128/web_final_Project`  
> 目標：將目前的校園專題組隊平台，從「功能型網站」升級成更有品牌感、更有互動性、更像正式產品的「校園專題媒合平台」。

---

## 一、目前網站狀態分析

### 1. 現有核心功能

目前網站已經具備完整的專題組隊平台雛形：

- 使用者登入 / 註冊
- 建立專題隊伍
- 首頁瀏覽隊伍
- 搜尋隊伍、課程、技能
- 依照狀態篩選隊伍
- 收藏隊伍
- 申請加入隊伍
- 檢舉隊伍
- 群組頁查看隊伍資訊
- 群組討論區
- 隊伍公告與倒數日期
- 邀請成員
- 審核加入申請
- 管理成員
- 轉移隊長
- 管理員處理檢舉與封鎖使用者

這表示專案已經不是單純展示型網站，而是有完整互動流程的 Web App。

### 改版主軸

本次改版不只是改顏色，而是要讓網站變成： ，注意原有的功能全部都要保留不能擅自刪除

- 後台儀表板

## 三、建議設計風格：Campus Glass UI

### 1. 視覺風格說明

目前網站已經有藍白風格，登入頁也有玻璃感與漸層背景。建議把這種風格延伸到全站。

新的風格可以命名為：

> Campus Glass UI

特色如下：

- 背景還是不要過多的顏色好了，漸層會顯得太多餘(以這句話為準，下面的背景修改不用理會)
- 卡片使用半透明白色
- 使用較大的圓角
- 使用柔和陰影
- 按鈕與輸入框更一致
- 減少過度放大的 hover 效果
- 強化狀態 badge 與技能 tag

## 2.4 script setup 新增 computed

```js
const openProjectCount = computed(() =>
  projects.value.filter((project) =>
    project.accepting_applications &&
    project.current_members < project.max_members
  ).length
)

const fullProjectCount = computed(() =>
  projects.value.filter((project) =>
    project.current_members >= project.max_members
  ).length
)

const topSkill = computed(() => {
  const countMap = {}

  projects.value.forEach((project) => {
    const skills = (project.required_skills || '')
      .split(',')
      .map((skill) => skill.trim())
      .filter(Boolean)

    skills.forEach((skill) => {
      countMap[skill] = (countMap[skill] || 0) + 1
    })
  })

  return Object.entries(countMap)
    .sort((a, b) => b[1] - a[1])?.[0]?.[0]
})
```

## 4.2 新增技能分數計算

```js
const matchPercent = computed(() => {
  const requiredSkills = (props.project.required_skills || '')
    .split(',')
    .map((skill) => skill.trim().toLowerCase())
    .filter(Boolean)

  const mySkills = (userSkills.value || '')
    .split(',')
    .map((skill) => skill.trim().toLowerCase())
    .filter(Boolean)

  if (!requiredSkills.length || !mySkills.length) {
    return 0
  }

  const matchedCount = requiredSkills.filter((skill) =>
    mySkills.includes(skill)
  ).length

  return Math.round((matchedCount / requiredSkills.length) * 100)
})
```

例如：

2. 再接到 Vue 畫面。
3. 最後測試登入、登出、權限狀態。

## 4. 權限一定要保留

推薦隊友、管理隊伍、刪除隊伍、Admin Dashboard 都必須確認權限。

尤其：

- 一般使用者不能進入 Admin
- 非隊長不能轉移隊長
- 副隊長不能刪除群組
- Admin 或擁有者才可以刪除群組
- 被封鎖使用者不能繼續操作
