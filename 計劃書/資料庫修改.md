# 資料庫修改.md

## 目標

本次修改目標是將專案中的使用者主鍵完全改為 `student_id`，並移除對 `users.id` 的依賴。

最終狀態：

```text
users.student_id = 使用者唯一主鍵
所有與使用者相關的外鍵都指向 users(student_id)
後端 API 不再查詢 users.id
前端仍維持原本功能，不因主鍵修改而壞掉
```

---

## 一、目前問題整理

目前資料庫的 `users` 表已經是：

```sql
student_id TEXT PRIMARY KEY
```

但是後端仍有部分程式碼使用舊邏輯：

```js
users.id
req.user.id
user.id
SELECT id FROM users
JOIN users ON users.id = ...
```

這會導致 SQLite 發生錯誤：

```text
no such column: id
```

因此 API 會回傳 `500 server error`，而不是正確的 `401 / 404 / 409`。

---

## 二、修改原則

### 1. 使用者主鍵統一為 student_id

之後所有使用者識別都使用：

```js
student_id
```

不要再使用：

```js
id
```

---

### 2. 資料庫欄位名稱可以先保留 user_id

部分資料表目前有：

```sql
user_id TEXT NOT NULL
```

雖然欄位名稱叫 `user_id`，但內容實際上應該存 `student_id`。

例如：

```sql
applications.user_id -> users.student_id
comments.user_id -> users.student_id
project_favorites.user_id -> users.student_id
notifications.user_id -> users.student_id
```

短期可以保留欄位名稱，避免一次改太大。

但程式邏輯要知道：

```text
user_id 欄位裡面存的是 student_id
```

---

### 3. 專題本身的 id 可以保留

這次只移除 `users.id`。

以下資料表仍可以保留自動編號 id：

```text
projects.id
applications.id
comments.id
notifications.id
project_invitations.id
project_announcements.id
project_deadlines.id
```

因為這些不是使用者主鍵，不需要一起改掉。

```sql
CREATE TABLE IF NOT EXISTS users (
  student_id TEXT PRIMARY KEY,
  username TEXT,
  name TEXT NOT NULL,

  email TEXT NOT NULL UNIQUE,
  password TEXT NOT NULL,
  role TEXT DEFAULT "user",
```


```sql
FOREIGN KEY(owner_id) REFERENCES users(student_id)

FOREIGN KEY(author_id) REFERENCES users(student_id)
FOREIGN KEY(inviter_id) REFERENCES users(student_id)
FOREIGN KEY(invitee_id) REFERENCES users(student_id)
```

### 檢查重點

```text

```


## 階段 2：修改 auth.js

檔案：

```text
server/middleware/auth.js
```

`signToken()` 應該統一使用 `student_id`。

建議保留 `id` 欄位作為前端過渡相容，但它的值也要等於 `student_id`：

```js
function signToken(user) {
  return jwt.sign(
    {
      id: user.student_id,
      student_id: user.student_id,
      email: user.email
    },
    JWT_SECRET,
    { expiresIn: '7d' }
  )
}
```

這樣：

```js
req.user.student_id
```

是正式使用方式。

```js
req.user.id
```

短期還能相容，但後端新程式不要再用它。

## 階段 3：修改 publicUser()

檔案：

```text
server/routes/api.js
```

原本可能是：

```js
function publicUser(row) {
  return {
    id: row.id,
    username: row.username || row.name,
    name: row.name,
    student_id: row.student_id,
    email: row.email
  }
}
```

要改成：

```js
function publicUser(row) {
  if (!row) {
    return null
  }

  return {
    id: row.student_id,
    student_id: row.student_id,
    username: row.username || row.name,
    name: row.name,
    email: row.email,
    role: row.role || 'user',
    avatar: row.avatar || '',
    department: row.department || '',
    grade: row.grade || '',
    skills: row.skills || '',
    bio: row.bio || '',
    github_url: row.github_url || '',
    is_suspended: row.is_suspended || 0,
    created_at: row.created_at
  }
}
```

### 為什麼 `id` 還要保留？

前端目前可能很多地方仍使用：

```js
user.id
```

如果後端直接拿掉 `id`，前端會壞。

所以先讓：

```text
user.id = user.student_id
```

這樣可以維持功能。

等所有前端都改成 `user.student_id` 後，再考慮移除 `id`。

```js

```

```js

  'SELECT student_id FROM users WHERE student_id = ?',
  [body.student_id]
```

### 4-3. 新增使用者後重新查詢

錯誤寫法：

```js
var result = await db.run(...)

```

因為 `users` 沒有 `id`，所以要改成：

```js
await db.run(
  'INSERT INTO users (username, name, student_id, email, password, role) VALUES (?, ?, ?, ?, ?, ?)',
  [body.name, body.name, body.student_id, body.email, hash, 'user']
)

var user = await db.get(
  'SELECT * FROM users WHERE student_id = ?',
  [body.student_id]
)
```

### 6-2. PUT /users/me

完整建議版本：

```js
router.put('/users/me', auth.authRequired, asyncHandler(async function(req, res) {
  var body = req.body
  var studentId = req.user.student_id

  var currentUser = await db.get(
    'SELECT student_id FROM users WHERE student_id = ?',
    [studentId]
  )

  if (!currentUser) {
    res.status(401).json({ message: '登入狀態已失效，請重新登入' })
    return
  }

  if (!required(body.email)) {
    res.status(400).json({ message: 'Email 為必填' })
    return
  }

  var email = String(body.email).trim()

  var exists = await db.get(
    'SELECT student_id FROM users WHERE email = ? AND student_id != ?',
    [email, studentId]
  )

  if (exists) {
    res.status(409).json({ message: 'Email 已被使用' })
    return
  }

  if (required(body.password)) {
    if (!/^[A-Za-z0-9]{6,}$/.test(body.password)) {
      res.status(400).json({
        message: '密碼至少 6 碼，且只能包含英文字母與數字'
      })
      return
    }

    var hash = await bcrypt.hash(body.password, 10)

    await db.run(
      'UPDATE users SET email = ?, password = ? WHERE student_id = ?',
      [email, hash, studentId]
    )
  }
  else{
    await db.run(
      'UPDATE users SET email = ? WHERE student_id = ?',
      [email, studentId]
    )
  }

  var user = await db.get(
    'SELECT * FROM users WHERE student_id = ?',
    [studentId]
  )

  res.json({ user: publicUser(user) })
}))
```


