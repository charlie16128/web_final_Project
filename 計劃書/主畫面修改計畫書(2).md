# 主畫面修改計畫書（SDD）

## 1. 文件目的

本文件規劃將目前主畫面整理成多頁式架構，但保留原本 `AppHeader`。  
修改重點是：

1. `AppHeader` 不移除、不改成 navbar
2. 在 `AppHeader` 下方新增一個 navbar
3. `/` 主畫面保留「申請搜尋」相關功能
4. navbar 內要有「主畫面」，點擊後回到 `/`
5. 其他功能拆到新頁面
6. 申請邀請頁面參考管理員頁面的切換方法，使用頁籤切換「我的申請 / 我的邀請」
7. 原本 style 盡量維持不變，只調整排版置中

---

## 2. 使用者需求修正

本次以以下需求為準：

- 保留 `AppHeader`
- 在 `AppHeader` 下方多加一個 navbar
- navbar 要有：
  - 主畫面
  - 我的群組
  - 申請邀請
  - 建立專題
- navbar 的「主畫面」要回到 `/`
- `/` 主畫面放「申請搜尋」
- 原本功能不要重新想，只把原本主畫面的功能拆開放
- hover navbar 文字時，文字下方要有一條線從左到右出現
- 各頁功能放在中間
- 申請邀請頁面要參考管理員頁面的切換方法顯示申請、邀請
- 使用 SDD 規劃

---

## 3. 目前狀況分析

目前 `HomeView.vue` 同時放了很多功能：

- `AppHeader`
- `GroupSidebar`
- `ProjectForm`
- 我的邀請
- 搜尋 toolbar
- `ProjectCard`
- 收藏、申請、檢舉、接受邀請、拒絕邀請等方法

目前 `AppHeader.vue` 負責顯示：

- TeamUp
- 校園專題組隊平台
- 使用者名稱
- 帳號設定
- 登出

所以這次不把 AppHeader 改掉，只在它下方新增 navbar。

            :class="{ active: activeTab === 'invitations' }"
            @click="activeTab = 'invitations'"

          </button>
        </div>

        <section v-if="activeTab === 'applications'" class="applications-list">
          <article v-if="!myApplications.length" class="mini-item">
            <p class="description">目前沒有申請紀錄。</p>
          </article>

            v-for="application in myApplications"

              <b>{{ application.project_title }}</b><br>

            <p class="description">目前沒有邀請。</p>

            v-for="invitation in myInvitations"
            :key="invitation.id"

            </button>

            </button>
          </div>
        </section>
      </section>
    </div>
  </main>

  <ToastMessage :message="toast" />
</template>

## Script 規劃

```js
import { onMounted, ref } from 'vue'

const activeTab = ref('applications')
const myApplications = ref([])
const myInvitations = ref([])

async function loadMyApplications() {
  const response = await api.get('/my-applications')
  myApplications.value = response.data.applications || []
}

async function loadMyInvitations() {
  const response = await api.get('/me/invitations')
  myInvitations.value = response.data.invitations || []
}

async function acceptInvitation(invitation) {
  try {
    await api.post(`/invitations/${invitation.id}/accept`)
    showToast('已接受邀請')
    await Promise.all([
      loadMyInvitations(),
      loadMyApplications()
    ])
  } catch (error) {
    showToast(error.response?.data?.message || '接受邀請失敗')
  }
}

async function rejectInvitation(invitation) {
  try {
    await api.post(`/invitations/${invitation.id}/reject`)
    showToast('已拒絕邀請')
    await loadMyInvitations()
  } catch (error) {
    showToast(error.response?.data?.message || '拒絕邀請失敗')
  }
}

onMounted(async () => {
  await Promise.all([
    loadUser(),
    loadMyApplications(),
    loadMyInvitations()
  ])
})
```
