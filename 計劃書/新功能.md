# 新功能.md

## 參考專案

專案：`charlie16128/web_final_Project`

目前專案採用前後端分離架構：

```text
client/  Vue 3 + Vite + Vue Router + Axios
server/  Express API + JWT auth + SQLite database
```

本文件使用 OpenSpec / SDD 的方式規劃，先拆清楚需求、資料庫、API、前端頁面與實作階段，避免一次大改造成錯誤難以追蹤。

---

# 一、整體目標

這次新增與修正的功能包含 6 個部分：

1. 群組新增「副隊長」權限
2. 管理員封鎖使用者後立即登出，封鎖時間精準到時分秒
3. 群組倒數日期改成可新增多個、橫向顯示、可查看詳細內容
4. 登入 / 註冊頁面背景與半透明 input 美化
5. input 錯誤訊息不再撐開版面
6. 全站原生 HTML 下拉選單改成自訂下拉選單元件

---

# 二、實作原則

## 2.1 不一次大改

每個功能都要分階段完成，完成一個階段後先確認畫面與 API 正常，再進入下一階段。

## 2.2 權限判斷統一放後端

前端可以控制按鈕顯示，但真正的權限限制一定要放在後端 API。

例如副隊長可以看到「踢出隊員」按鈕，但後端仍然要檢查這個人是不是隊長或副隊長，不能只相信前端。

## 2.3 共用元件優先

登入註冊背景、input 錯誤訊息、自訂下拉選單都屬於全站樣式，應盡量做成共用 CSS 或共用 Vue component，避免每個頁面重複寫。

# 三、功能 1：群組新增副隊長

## 3.1 需求說明

目前群組應該只有隊長可以管理隊伍。這次要新增「副隊長」角色。

副隊長可以：

- 在 group 的隊伍管理中編輯 group 資訊
- 邀請成員
- 踢出隊員


副隊長不可以：

- 解散群組
- 轉移隊長
- 把自己或其他人設定為隊長
- 修改隊長權限

隊長可以：

- 設定成員為副隊長
- 取消副隊長
- 轉移隊長
- 編輯 group 資訊
- 邀請成員
- 踢出隊員
- 解散群組

一般成員只可以：

- 查看群組資訊
- 查看成員
- 參與群組討論

---

## 3.2 資料庫規劃

### 方案 A：如果目前 group_members 已經有 role 欄位

直接擴充 role 值：

# 四、功能 2：封鎖後立即登出 + 封鎖時間精準到時分秒

## 4.1 需求說明

目前管理員封鎖使用者後，該使用者應該立即失去登入狀態。

封鎖時間也要從原本可能只支援天數，改成支援：

```text
天 / 小時 / 分鐘 / 秒
```

例如：

```text
封鎖 1 天 2 小時 30 分 0 秒
封鎖 3 小時
封鎖 10 分鐘
```

---

## 4.2 資料庫規劃

users 表建議新增或確認欄位：

```sql
ALTER TABLE users ADD COLUMN banned_until TEXT;
ALTER TABLE users ADD COLUMN token_version INTEGER DEFAULT 0;
```

### banned_until

用 ISO time 或 SQLite datetime 字串儲存。

例如：

```text
2026-06-17T18:30:00.000Z
```

如果 `banned_until` 大於現在時間，代表仍被封鎖。

### token_version

用來讓舊 JWT 立即失效。

當管理員封鎖使用者時：

```sql
UPDATE users
SET banned_until = ?, token_version = token_version + 1
WHERE id = ?;
```

這樣該使用者原本存在 localStorage 的 token 會失效。

---

## 4.3 JWT 修改

登入成功簽發 token 時，要把 token_version 放進 payload：

```js
jwt.sign({
    id: user.id,
    role: user.role,
    tokenVersion: user.token_version
}, JWT_SECRET)
```

驗證 token 時：

1. 從 token 取出 userId 與 tokenVersion
2. 查資料庫目前 token_version
3. 如果不一致，回傳 401
4. 如果 banned_until 還沒過，也回傳 403 或 401

建議回傳格式：

```json
{
  "message": "帳號已被封鎖",
  "banned_until": "2026-06-17T18:30:00.000Z"
}
```

---

## 4.4 前端立即登出處理

在 Axios interceptor 統一處理：

```js
axios.interceptors.response.use(
    response => response,
    error => {
        if(error.response?.status === 401 || error.response?.status === 403){
            const message = error.response.data?.message
            if(message?.includes('封鎖') || message?.includes('登入已失效')){
                localStorage.removeItem('token')
                localStorage.removeItem('user')
                router.push('/login')
            }
        }
        return Promise.reject(error)
    }
)
```

這樣使用者只要下一次呼叫任何 API，就會被登出。

如果希望「完全立即」，需要 WebSocket 或輪詢，目前專案是 Express API 架構，建議先用 API token_version 方式，穩定又比較容易完成。

---

## 4.5 管理員封鎖表單

原本只輸入天數的地方，改成四個欄位：

```text
天：number
小時：number
分鐘：number
秒：number
```

前端送出：

```json
{
  "days": 1,
  "hours": 2,
  "minutes": 30,
  "seconds": 0
}
```

後端計算：

```js
const totalMs =
    days * 24 * 60 * 60 * 1000 +
    hours * 60 * 60 * 1000 +
    minutes * 60 * 1000 +
    seconds * 1000
```

限制：

- 四個欄位不可全部為 0
- 不可為負數
- 小時建議 0~23
- 分鐘建議 0~59
- 秒數建議 0~59

# 七、功能 5：input 錯誤訊息不撐開版面

## 7.1 問題說明

目前錯誤訊息顯示在 input 框下面，當錯誤訊息出現時，會把下方物件往下推，導致整個 div 高度改變。

這會讓畫面跳動，看起來不穩定。

---

## 7.2 解法一：保留錯誤訊息高度

每個 input 外層固定預留錯誤訊息高度。

```vue
<div class="form-field">
    <input class="auth-input" />
    <p class="field-error">{{ errorMessage }}</p>
</div>
```

```css
.form-field{
    margin-bottom: 14px;
}

.field-error{
    min-height: 18px;
    margin: 4px 0 0;
    font-size: 12px;
    color: #ef4444;
    line-height: 18px;
}
```

優點：

- 最穩定
- 實作簡單
- 不會造成版面跳動

建議優先使用這個方案。

---

## 7.3 解法二：錯誤訊息使用 absolute

```css
.form-field{
    position: relative;
    margin-bottom: 22px;
}

.field-error{
    position: absolute;
    left: 2px;
    bottom: -18px;
    font-size: 12px;
    color: #ef4444;
}
```

優點：

- 不會撐開 div

缺點：

- 如果錯誤訊息太長，可能會蓋到其他元素
- 每個欄位間距要控制好

---

## 7.4 最終建議

登入 / 註冊頁使用解法一。

其他表單如果空間比較小，可以使用解法二。

共用 class：

```text
.form-field
.field-error
.input-error
```

input 錯誤狀態：

```css
.input-error{
    border-color: rgba(239, 68, 68, 0.9);
    box-shadow: 0 0 0 4px rgba(239, 68, 68, 0.12);
}
```

# 八、功能 6：全站自訂下拉選單

## 8.1 需求說明

需求：

- 不使用原生 HTML 下拉式選單外觀
- 要符合網站風格
- 要有 border radius
- 支援選項列表
- 支援 placeholder
- 支援點擊展開 / 收合
- 點擊外部自動關閉
- 選到項目後更新 v-model

---

## 8.2 建議新增元件


options 格式：

## 8.4 CustomSelect.vue 規劃

```vue
<template>
    <div class="custom-select" ref="selectRef">
        <button class="select-trigger" type="button" @click="toggleOpen">
            <span>{{ selectedLabel || placeholder }}</span>
            <span class="select-arrow" :class="{ open: isOpen }">⌄</span>
        </button>

        <div v-if="isOpen" class="select-menu">
            <button
                v-for="option in options"
                :key="option.value"
                type="button"
                class="select-option"
                :class="{ active: option.value === modelValue }"
                @click="selectOption(option)"
            >
                {{ option.label }}
            </button>
        </div>
    </div>
</template>
```

```js
<script setup>
import { computed, onBeforeUnmount, onMounted, ref } from 'vue'

const props = defineProps({
    modelValue: [String, Number, Boolean],
    options: {
        type: Array,
        default: () => []
    },
    placeholder: {
        type: String,
        default: '請選擇'
    }
})

const emit = defineEmits(['update:modelValue'])
const isOpen = ref(false)
const selectRef = ref(null)

const selectedLabel = computed(() => {
    return props.options.find(option => option.value === props.modelValue)?.label || ''
})

function toggleOpen(){
    isOpen.value = !isOpen.value
}

function selectOption(option){
    emit('update:modelValue', option.value)
    isOpen.value = false
}

function handleClickOutside(event){
    if(selectRef.value && !selectRef.value.contains(event.target)){
        isOpen.value = false
    }
}
```

```sql
-- group members role

-- user ban and token invalidation
ALTER TABLE users
ADD COLUMN banned_until TEXT;

ALTER TABLE users
ADD COLUMN token_version INTEGER DEFAULT 0;

-- group countdowns
CREATE TABLE IF NOT EXISTS group_countdowns (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_id INTEGER NOT NULL,
    title TEXT NOT NULL,
    description TEXT,
    target_time TEXT NOT NULL,
    created_by INTEGER NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

注意：如果欄位已存在，不要重複 ALTER TABLE，否則 SQLite 會報錯。實作時要先檢查欄位是否存在，或手動確認資料庫結構。
