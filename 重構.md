# TeamUp Campus 專案重構計畫

> 目標：在**不破壞現有功能**的前提下，將目前的「Express + Vue CDN + SQLite」專案，逐步重構成老師要求的：
>
> ```text
> /project
>   /server   ← Express（API only）
>   /client   ← Vue + Vite
> ```

---

## 一、目前專案狀態

目前專案大致使用：

- 前端：HTML、CSS、JavaScript、Vue 3 CDN
- 後端：Express
- 資料庫：SQLite
- 驗證：JWT、bcrypt
- 頁面由 Express 提供
- API 路徑統一放在 `/api`

目前 `app.js` 同時負責：

1. 啟用 Express middleware
2. 提供 `public` 靜態檔案
3. 掛載 `/api`
4. 掛載頁面路由

目前頁面路由包含：

```text
/
 /login
 /register
 /groups/:id
```

這些路由會回傳 `public` 目錄中的 HTML。

目前後端 API 已經包含：

- 註冊
- 登入
- 取得目前使用者
- 編輯帳號
- 查詢專案
- 建立專案
- 編輯專案
- 刪除專案
- 申請加入專案
- 審核申請
- 查詢群組
- 群組留言
- 專案留言
- 查詢我的申請

因此這次重構的重點不是重寫後端功能，而是：

1. 將 Express 移入 `/server`
2. 建立獨立的 Vue + Vite `/client`
3. 將原本 HTML + Vue CDN 頁面逐頁搬成 `.vue`
4. 保留既有 API 路徑與資料庫
5. 最後才移除舊 `public` 頁面

---

# 二、重構原則

## 2.1 不一次全部重寫

禁止直接刪除：

```text
public/
routes/pages.js
原本的 main.js
原本的 HTML
```

在新 Vue 前端尚未完成前，這些檔案都是可運作的備援版本。

---

## 2.2 保持 API 格式不變

重構前端時，應優先配合現有 API，不要同時修改前端與後端資料格式。

例如目前登入 API：

```http
POST /api/login
```

回傳：

```json
{
  "token": "...",
  "user": {}
}
```

新 Vue 前端仍先沿用這個格式。

不要在第一階段把它改成：

```http
POST /api/auth/login
```

否則會增加不必要的錯誤風險。

---

## 2.3 保留 SQLite 資料

重構期間：

- 不刪除資料庫
- 不重建資料表
- 不修改欄位名稱
- 不修改既有 SQL
- 不搬動資料庫後忘記更新路徑

若要搬動資料庫，必須先確認 `database/db.js` 使用的是絕對路徑或以 `__dirname` 組合的路徑。

---

## 2.4 每一階段都必須可以執行

每完成一個階段，就要測試：

- 後端是否可以啟動
- 舊前端是否仍可使用
- 新前端是否可以啟動
- API 是否正常
- SQLite 是否仍可讀寫

## 5.4 檢查資料庫路徑

`server/database/db.js` 不應依賴執行指令所在位置。

不建議：


```js

const db = new sqlite3.Database(dbPath)
```

原因是之後從專案根目錄執行指令時，相對路徑可能改變。

## 6.1 設定 Vite Proxy

修改：

```js
// client/vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true
      }
    }
  }
})
```

這樣 Vue 可以呼叫：

```js
fetch('/api/projects')
```

而不是：

```js
fetch('http://localhost:3000/api/projects')
```

---

## 6.2 建立 API 模組

建立：

```js
// client/src/services/api.js
import axios from 'axios'

const api = axios.create({
  baseURL: '/api',
  timeout: 10000
})

api.interceptors.request.use((config) => {
  const token = localStorage.getItem('teamup_token')

  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }

  return config
})

api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('teamup_token')
    }

    return Promise.reject(error)
  }
)

export default api
```

不要在每個元件中重複撰寫 Authorization header。

# 七、第三階段：建立 Vue Router

建立：

```js
// client/src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'

import HomeView from '../views/HomeView.vue'
import LoginView from '../views/LoginView.vue'
import RegisterView from '../views/RegisterView.vue'
import GroupView from '../views/GroupView.vue'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      name: 'home',
      component: HomeView,
      meta: {
        requiresAuth: true
      }
    },
    {
      path: '/login',
      name: 'login',
      component: LoginView
    },
    {
      path: '/register',
      name: 'register',
      component: RegisterView
    },
    {
      path: '/groups/:id',
      name: 'group',
      component: GroupView,
      meta: {
        requiresAuth: true
      }
    }
  ]
})

router.beforeEach((to) => {
  const token = localStorage.getItem('teamup_token')

  if (to.meta.requiresAuth && !token) {
    return {
      name: 'login'
    }
  }

  if (
    token &&
    (to.name === 'login' || to.name === 'register')
  ) {
    return {
      name: 'home'
    }
  }
})

export default router
```

修改：

```js
// client/src/main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import './assets/style.css'

createApp(App)
  .use(router)
  .mount('#app')
```

修改：

```vue
<!-- client/src/App.vue -->
<template>
  <RouterView />
</template>
```

## 8.1 LoginView.vue

將舊 `login.html` 中 `#app` 裡的內容搬進：

```vue
<template>
  <main class="auth-page">
    <!-- 舊登入畫面 -->
  </main>
</template>

<script setup>
import { reactive, ref } from 'vue'
import { useRouter } from 'vue-router'
import api from '../services/api'

const router = useRouter()
const loading = ref(false)
const errorMessage = ref('')

const form = reactive({
  email: '',
  password: ''
})

async function login() {
  errorMessage.value = ''
  loading.value = true

  try {
    const response = await api.post('/login', form)

    localStorage.setItem(
      'teamup_token',
      response.data.token
    )

    await router.replace({
      name: 'home'
    })
  } catch (error) {
    errorMessage.value =
      error.response?.data?.message ||
      '登入失敗，請稍後再試'
  } finally {
    loading.value = false
  }
}
</script>
```

注意：目前後端登入 API 是：

```text
POST /api/login
```

因此 `api.post('/login')` 才是正確寫法。

```text
[ ] CSS 搬移完成
[ ] Vue Router 可以處理所有前端路由
```

---

## 13.1 修改 server/app.js

原本：

```js
var path = require('path')
var pageRouter = require('./routes/pages')

app.use(
  express.static(
    path.join(__dirname, 'public'),
    { index: false }
  )
)

app.use('/api', apiRouter)
app.use('/', pageRouter)
```

改成：

```js
var express = require('express')
var cookieParser = require('cookie-parser')
var logger = require('morgan')
var cors = require('cors')

var apiRouter = require('./routes/api')

var app = express()

app.use(logger('dev'))
app.use(cors({
  origin: process.env.CLIENT_ORIGIN ||
    'http://localhost:5173'
}))
app.use(express.json())
app.use(express.urlencoded({
  extended: false
}))
app.use(cookieParser())

app.use('/api', apiRouter)

module.exports = app
```

這時 Express 才正式成為 API only。

```js
app.use('/api', function(req, res) {
```

# 十八、常見錯誤與避免方式

## 18.1 API 資料格式寫錯

目前：

```json
{
  "projects": []
}
```

不要誤寫：

```js
response.data.data
```

應使用：

```js
response.data.projects
```

---

## 18.2 Token 名稱改掉

舊版使用：

```text
teamup_token
```

新版繼續使用相同名稱：

```js
localStorage.getItem('teamup_token')
```

避免使用者登入狀態突然失效。

---

## 18.3 SQLite 路徑錯誤

搬到 `server` 後，檢查：

```js
path.join(__dirname, '檔名.sqlite')
```

不要依賴 `process.cwd()`。

## 18.4 Express 先刪 public

新前端完成前不能刪除：

```text

server/routes/pages.js
```

## 18.5 同時修改所有 API 路徑

```text

```

改成：

```text
/api/auth/login
```

先完成架構重構，再做 API 整理。

## 18.6 Vue Router 重新整理 404

開發環境由 Vite 處理。

部署時必須設定 SPA fallback，讓：

```text
/groups/5
```

重新整理時仍回傳 `index.html`。

# 二十二、最重要的保護規則

重構過程中始終遵守：

```text
舊版可以跑
    ↓
建立新版的一小部分
    ↓
測試新版
    ↓
測試舊版
    ↓
Commit
    ↓
再進行下一部分
```

只要某個階段出錯，就回到上一個 commit，不要在錯誤狀態下繼續往後改。

最終確認 Vue + Vite 版本所有功能都正常後，才刪除舊版 Vue CDN 前端。
